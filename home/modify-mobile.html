# 移动端修改点

# 修改时间

2019.9.6

  1. 通用样式 common、 reset、 muse-ui-reset 统一用UI组件库的。

  2. 国际化移到UI组件库里，因为UI组件库里也有不少国际化相关的代码，统一维护。

  3. 模块或组件里的 headbar 组件统一去除，去掉相关代码，如果有钉钉的bar的代码也对应删除。

  4. 新增了 initAppId 函数，根据 json文件来初始化 body 的 ID：

    Version: alphaflow 极其衍生版本 -- id为 af_mobile

    Version: dingding 极其衍生版本 -- id为 af_mobile_dingding

    注意

    默认该版本下，header-bar 是置顶的，也就是默认用dingding app里的bar

  5. 样式引用时候注意，老代码里是会把 common.less 引入，代码拿过来的时候记得改掉，现在版本使用方式：

    /* components.less */
@import "../../../assets/styles/ui-config.less";

    ui-config.less 已经包含所有的方法和变量，包括钉钉和af的，可以直接使用。

  6. 删掉了一些UI组件库里重复的组件。

  7. 基础版本里把头和模块分离处理，对应的事件和描述信息单独处理，解耦：

    <template>
  <div>
    <AfHeadBar :titleText="$t('message.loginPwd')" :backCb="backCb" />
    <AfPwdLogin />
  </div>
</template>

<script>
import { Component, Prop, Vue } from "vue-property-decorator";
import { AfHeadBar } from "afmobileui";
import { AfPwdLogin } from "afmobilepro";
import { routerGo } from "@/util/tool";
Vue.use(AfPwdLogin);
Vue.use(AfHeadBar);
@Component
export default class PwdLogin extends Vue {
  backCb() {
    routerGo({
      instance: this,
      url: "/"
    });
  }
}
</script>
    1. 业务组件库或者UI组件库里，不要用 @/ 这种绝对路径的形式来import，原因是装在base版本里的时候，webpack解析不了。
